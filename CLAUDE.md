# CLAUDE.md

Guidance for Claude Code working in this repo.

## What this project does

Measures the lag between Chromium security fixes and end users of Electron
apps. Each Electron release pins a Chromium version; app maintainers must
manually upgrade. This repo tracks which Electron version each known app
ships and cross-references against Electron's stable release dates.

## Run things

Local Python is managed with `uv` — always invoke commands as
`uv run main.py <command>`. Python ≥ 3.14 (see `pyproject.toml`).

Pipeline (each step writes back to `data/apps/`):

    uv run main.py extract-apps        # repos/electron-apps → data/apps/
    uv run main.py fetch-versions      # npm electron releases → data/versions.txt
    uv run main.py process github.com  # latest releases + asset URLs
    uv run main.py process source      # download src archives, detect electron
    uv run main.py process which-electron  # binary fingerprint via captn3m0/which-electron (opt-in)
    uv run main.py process --aur       # AUR package match (opt-in)
    uv run main.py stats
    uv run main.py report              # writes REPORT.md

Set `GITHUB_TOKEN` (or `GH_TOKEN`) for the `github.com` processor.

External meta files (AUR index, Homebrew casks, Electron headers) are pulled
by `make all` into `meta/`.

## Layout

- `main.py` — Click entry point; just calls `commands.cli`.
- `commands/` — one Click subcommand per file, registered via
  `commands/__init__.py`. Shared helpers (`load_apps`, `write_app`,
  `DATA_DIR`, `APPS_DATA_DIR`) also live there.
- `steps/` — auto-discovered processors. The filename is treated as a
  domain (e.g. `github.com.py`); a processor matches an entry when its
  `repository` URL has that netloc, unless the module exports a custom
  `matches(entry)` callable. Set `AUTO = False` to opt out of default runs
  (e.g. `aur.py`, `homebrew.py`).
- `processors/` — non-step helpers (currently just the npm fetcher).
- `repos/electron-apps/` — submodule, the upstream app registry.
- `meta/` — locally cached external indexes (built by `make`).
- `zips/`, `src/` — download / extraction caches; not committed.

## Data files

All app data lives in `data/apps/`, one YAML file per app, named `<id>.yml`.
Each file is a dict with `id` and optional `name`, `repository`, `website`,
`downloads`, `packages`, `aur`, `homebrew`, `electron`, `method`, `src`,
`latest`, `dead`, etc. Loaded via `commands.load_apps()` and written via
`commands.write_app(entry)`. `data/apps.yml` is the legacy single-file form
that `load_apps()` migrates from on first call if `data/apps/` is missing.

Two flavours of download list coexist on the same entry:

- `downloads` — `[{url, name, version?, checksum?}]`. Populated by the
  `github.com` and `static-downloads` processors and used by the `source`
  processor.
- `packages` — `[{url, format, os, arch?}]`. Manually curated stable
  download URLs (formerly `data/zoo-extra.yml`) for apps not covered by
  GitHub releases or whose installers live off-GitHub. When any package
  URL points at github.com, the entry gets a top-level `repository:` so
  the `github.com` processor can detect the bundled Electron version.

Other data:

- `data/versions.txt` — tab-separated `<version>\t<ISO date>` for every
  stable Electron release (excluding nightly/alpha).
- `REPORT.md` — generated by `uv run main.py report`.

## Conventions

- Per-app YAML is written with `default_flow_style=False`, `sort_keys=False`,
  unicode preserved. Always go through `write_app()` so writes stay atomic.
- Processors return a partial dict (or `None`) and the runner merges it
  into the entry; processors must not mutate `entry` directly.
- `repository:` is the canonical "this lives on host X" key — the
  `github.com` processor keys off it. Adding a `repository:` URL is
  enough to bring an entry into the GitHub release / source-detection
  pipeline.
- Entries marked `dead: true` are skipped by `process`.
- The `source` processor's preference order for detecting an electron
  version is: `package-lock.json` > `yarn.lock` > `pnpm-lock.yaml` >
  `package.json` (range resolved against `data/versions.txt`).

## Don't

- Don't run Python directly — always `uv run`.
- Don't hand-edit `data/apps/*.yml` for fields the processors own
  (`downloads`, `electron`, `method`, `latest`, `src`); they'll be
  overwritten on the next `process` run. `repository`, `website`,
  `name`, `dead`, and the opt-out flags (`aur: false`, `homebrew: false`)
  are safe to set manually.
- Don't commit `zips/` or `src/`.
